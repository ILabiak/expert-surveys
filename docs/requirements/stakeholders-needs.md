# Запити зацікавлених осіб

## Вступ

*Цей розділ надає ознайомчу інформацію для зацікавлених у використанні цього сервісу осіб, які можуть бути як викладачами та студентами, так і менеджерами та аналітиками, для яких необхідна організація опитувань спеціалістів, збір та аналіз інформації.*

### Мета

*Створення сучасного, зручного та гнучкого сервісу опитування експертів, який дозволить його користувачам збирати та аналізувати інформацію ефективно та продуктивно.*

### Контекст

Цей документ формує технічне завдання для проекту системи організації та управління опитуваннями експертів "Expert Surveys".

## <a name="1">Характеристика ділових процесів</a>

Функціональність сервісу опитувань призначена для надання послуг з організації повного циклу експертного опитування. Результатом роботи сервісу є отримання клієнтом зібраних даних. Клієнти можуть в будь-який момент зв’язатися з сервіс-менеджером, що обслуговує  користувачів через систему технічної підтримки для отримання необхідної допомоги у вирішенні виниклих питань.

### Сценарії використання
 #### Сценарій №1  
**ID:** BC-1  
**НАЗВА:** Запит на створення нового опитування  
**УЧАСНИКИ:** Аналітик, Менеджер, Замовник  
**ПЕРЕДУМОВИ:** Немає  
**РЕЗУЛЬТАТ:** Опитування успішно додане  
**ВИКЛЮЧНІ СИТУАЦІЇ:**  
  -	EX-1: Аналітик не схвалив додання нового опитування  
  
**ОСНОВНИЙ СЦЕНАРІЙ:**  
1. Замовник передає менеджеру заявку на опитування.  
2. Менеджер реєструє замовлення.  
3. Менеджер повідомляє аналітику про нове замовлення.  
4. Аналітик аналізує доцільність реалізації замовлення (можлива ситуація EX-1).  
5. Аналітик повідомляє результат аналізу менеджеру.  
6. Менеджер повідомляє результат замовнику.  

#### Сценарій №2  
**ID:** BC-2  
**НАЗВА:** Редагування опитування  
**УЧАСНИКИ:** Аналітик, Менеджер, Замовник  
**ПЕРЕДУМОВИ:** Наявність опитування  
**РЕЗУЛЬТАТ:** Опитування успішно скориговане  
**ВИКЛЮЧНІ СИТУАЦІЇ:**  
  -	EX-2: Аналітик не схвалив зміни  
  
**ОСНОВНИЙ СЦЕНАРІЙ:**  
1. Замовник повідомляє менеджеру про бажання зробити зміни в опитуванні.  
2. Менеджер повідомляє аналітику про бажання замовника внести зміни.  
3. Аналітик переконується в коректності та доцільності змін (можлива ситуація EX-2).  
4. Аналітик повідомляє результат аналізу менеджеру.  
5. Менеджер підтверджує зміни, або, у випадку, EX-2 повідомляє причину скасування.  

#### Сценарій №3  
**ID:** BC-3  
**НАЗВА:** Перегляд статистики опитування  
**УЧАСНИКИ:** Замовник  
**ПЕРЕДУМОВИ:** Наявність результатів опитування  
**РЕЗУЛЬТАТ:** Отримання статистики  
**ВИКЛЮЧНІ СИТУАЦІЇ:** Відсутні  
**ОСНОВНИЙ СЦЕНАРІЙ:** Замовник переглядає статистику опитування  

#### Сценарій №4  
**ID:** BC-4  
**НАЗВА:** Надсилання запрошення для участі в опитуванні  
**УЧАСНИКИ:** Менеджер, Система  
**ПЕРЕДУМОВИ:** Наявність опитування  
**РЕЗУЛЬТАТ:** Експерт отримав запрошення  
**ВИКЛЮЧНІ СИТУАЦІЇ:** Відсутні  
**ОСНОВНИЙ СЦЕНАРІЙ:**  
1. Менеджер обирає експертів, що будуть брати участь в опитуванні
2. Система надсилає запрошення  

#### Сценарій №5  
**ID:** BC-5  
**НАЗВА:**  Отримання результату проходження опитування  
**УЧАСНИКИ:** Система, Експерт  
**ПЕРЕДУМОВИ:** Експерт заповнив анкету опитування  
**РЕЗУЛЬТАТ:** Система зберегла результат проходження опитування  
**ВИКЛЮЧНІ СИТУАЦІЇ:** Відсутні  
**ОСНОВНИЙ СЦЕНАРІЙ:**  
1. Експерт підтверджує відправлення анкети.  
2. Система отримує і зберігає результат опитування.  

#### Сценарій №6  
**ID:** BC-6  
**НАЗВА:** Нагадування про наближення закінчення терміну проведення опитування  
**УЧАСНИКИ:** Менеджер  
**ПЕРЕДУМОВИ:** Експерт не пройшов опитування. До закінчення терміну проведення опитування залишилась певна кількість часу (в залежності від опитування)  
**РЕЗУЛЬТАТ:** Експерт отримав повідомлення з нагадуванням  
**ВИКЛЮЧНІ СИТУАЦІЇ:** Відсутні  
**ОСНОВНИЙ СЦЕНАРІЙ:** Менеджер надсилає експерту повідомлення з нагадуванням про закінчення терміну проведення опитування  


## <a name ="2">Короткий огляд продукту</a>

**Expert Surveys** - система організації та управління опитуваннями експертів, що полегшує взаємодію між зацікавленими особами та експертами. Доступ до платформи здійснюється через web-сторінку в мережі Інтернет. В подальшому планується створення додатків для операційних систем Android та IOS. 


## <a name="3">Функціональність</a>

Вимоги до функціональності залежить від катергорії зацікавленої особи. 

#### Категорії зацікавлених осіб 
* Менеджер опитування 
* Аналітик
* Експерт

#### Функціональність користувача
* Рееєстрація
* Авторизація 

#### Вимоги до функциональності для особи "Менеджер опитування" extends користувач
* Керувати процесом підготовки та проведення опитування. 
* Надсилати запрошення для участі в опитуванні;  
* Формувати запит на створення опитування аналітиком.

#### Вимоги особи "Аналітик" extends користувач
* Створювати опитування на замовлення "Замовника";  
* Додавати питання в опитування. 

#### Вимоги особи "Експерт" extends користувач
* Проходити опитування, створеного особою "Аналітик"


## <a name="4">Практичність</a>

Основними вимогами до практичності предмета розробки є:
- Інтерфейс програмного продукту має бути зрозумілим і зручним для роботи користувача.
- Предмет розробки має бути оптимізованим під різні мови.
- Програмний продукт має бути зручним для використання як на комп’ютері, так і на мобільних пристроях.


## <a name="5">Надійність</a>

Основними вимогами до надійності предмета розробки є:
- Система має підтримувати стабільність роботи при обслуговуванні великої кількості користувачів і приладів.
- Система має здійснювати резервне копіювання баз даних для запобігання їх загублення.


## <a name="6">Продуктивність</a>

* Система може стикатися із великими навантеженнями, адже опитування будуть проходити одночасно велика кількість людей (~1000 людей/добу).

* В опитуваннях можуть бути графічні зображення, тож треба виділяти на них достатню кількість пам'яті (~1 мб/картинка)

## <a name="7">Експлуатаційна придатність</a>

* Кросплатформенність, тобто одночасна підтримка різних девайсів. Наприклад, мобільний телефон на операційній системі Android або персональний комп'ютер на Windows.
